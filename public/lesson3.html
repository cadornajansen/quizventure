LESSON 3: DECLARING & INITIALIZING VARIABLES

Lesson Summary

This lesson dives into declaring and initializing variables in Java, building on the basics of variables. Declaration introduces a variable to the compiler with its type and name, while initialization assigns an initial value. You'll learn the syntax, best practices, and how to handle different scenarios like multiple declarations or default values. Practical examples and exercises will help you master setting up variables for dynamic programs.

Lesson 3: Declaring & Initializing Variables

Declare a variable to tell Java its type and name, then initialize it with a value. It's like setting up your game character! Declaration reserves memory and defines the variable's properties, while initialization gives it a starting value. This two-step process ensures your code is clear and error-free.

Declaration Syntax

Use the data type followed by the variable name. No value is assigned yet.

// Basic declaration
int count;

// Declaration with multiple variables
int x, y, z;

Declarations must happen before use, and variables are case-sensitive.

Initialization Syntax

Assign a value using the equals sign (=). This can be done separately or combined with declaration.

// Separate steps
int count;
count = 10;

// Combined
double price = 99.99;

// Multiple initializations
int a = 1, b = 2, c = 3;

Challenge: Try declaring a String variable for your favorite game.

Default Values and Uninitialized Variables

If not initialized, local variables have no default value (compile error if used). Instance variables default to 0, false, or null based on type. Always initialize for safety!

Best Practices

Declare variables close to use, use descriptive names, and initialize immediately when possible. Avoid global variables unless necessary.

Examples and Code Snippets

See declaration and initialization in action.

// Example setup
String playerName;
int playerScore;

playerName = "Hero";
playerScore = 1500;

System.out.println(playerName + " has " + playerScore + " points.");

Try It Yourself: Declaration & Initialization Exercises

Run these in a Java environment to practice.

Exercise 1: Basic Declaration and Initialization

public class DeclareInitDemo {
  public static void main(String[] args) {
    // Declare
    String itemName;
    double itemPrice;
    int itemQuantity;

    // Initialize
    itemName = "Potion";
    itemPrice = 25.50;
    itemQuantity = 5;

    // Output
    System.out.println("Item: " + itemName);
    System.out.println("Price: $" + itemPrice);
    System.out.println("Quantity: " + itemQuantity);
    System.out.println("Total: $" + (itemPrice * itemQuantity));
  }
}
// Output:
// Item: Potion
// Price: $25.5
// Quantity: 5
// Total: $127.5

Exercise 2: Combined Declaration and Initialization

public class CombinedDemo {
  public static void main(String[] args) {
    // Combined
    String gameMode = "Adventure";
    int levelsCompleted = 3;
    boolean hasPowerUp = true;

    // Modify
    levelsCompleted += 2;  // Increment
    hasPowerUp = false;

    System.out.println("Mode: " + gameMode);
    System.out.println("Levels: " + levelsCompleted);
    System.out.println("Power-up: " + hasPowerUp);
  }
}
// Output:
// Mode: Adventure
// Levels: 5
// Power-up: false

Common Pitfalls

Watch for uninitialized use (causes errors) and type mismatches. Use final for constants.

Pro Tip: In games, initialize player stats at the start for smooth gameplay.
