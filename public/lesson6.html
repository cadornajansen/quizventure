Lesson 6: Switch & Ternary Operator

Switches handle multiple choices efficiently, while the ternary operator is a shorthand for simple if-else. They provide alternatives to long if-else chains, improving readability and performance. Switches work best with discrete values, while ternaries are for quick assignments.

Switch Statement

Evaluates an expression and matches it to cases. Use break to exit; default handles unmatched values.

switch (expression) {
  case value1:
    // Code
    break;
  case value2:
    // Code
    break;
  default:
    // Default code
}

Supports int, char, String (Java 7+), and enums. No fall-through without break.

Ternary Operator

Shorthand for if-else: condition ? trueValue : falseValue. Returns a value, not executes code.

variable = (condition) ? valueIfTrue : valueIfFalse;

Great for assignments; avoid nesting for clarity.

When to Use Each

  • Switch: Multiple fixed options (e.g., menu selections).
  • Ternary: Simple conditionals in assignments.

Power-Up: Use switches for menu selections in games!

Examples and Code Snippets

Practical uses.

// Ternary example
int max = (a > b) ? a : b;

// Switch example
switch (day) {
  case 1: System.out.println("Monday"); break;
  default: System.out.println("Another day");
}

Try It Yourself: Switch & Ternary Exercises

Experiment with these.

Exercise 1: Switch Statement

public class SwitchDemo {
  public static void main(String[] args) {
    int level = 2;

    switch (level) {
      case 1:
        System.out.println("Beginner level");
        break;
      case 2:
        System.out.println("Intermediate level");
        break;
      case 3:
        System.out.println("Advanced level");
        break;
      default:
        System.out.println("Unknown level");
    }

    // String switch (Java 7+)
    String fruit = "apple";
    switch (fruit) {
      case "apple":
        System.out.println("Red fruit");
        break;
      case "banana":
        System.out.println("Yellow fruit");
        break;
      default:
        System.out.println("Other fruit");
    }
  }
}
// Output:
// Intermediate level
// Red fruit

Exercise 2: Ternary Operator

public class TernaryDemo {
  public static void main(String[] args) {
    int score = 85;
    String result = (score >= 60) ? "Pass" : "Fail";
    System.out.println("Result: " + result);

    // Nested ternary (use sparingly)
    int num = 10;
    String desc = (num > 0) ? "Positive" : (num < 0) ? "Negative" : "Zero";
    System.out.println("Number is: " + desc);

    // In assignment
    double discount = (score > 90) ? 0.1 : 0.05;
    System.out.println("Discount: " + (discount * 100) + "%");
  }
}
// Output:
// Result: Pass
// Number is: Positive
// Discount: 5.0%

Common Pitfalls and Tips

Forget break in switches causes fall-through. Ternaries can be hard to read if complex. Test all cases. In games, switches handle character classes or item types.

Pro Tip: Prefer switches over long if-else for performance on many options.
