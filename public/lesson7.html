Lesson 7: OOP Concepts

Object-Oriented Programming (OOP) treats code like real-world objects. It's the backbone of Java and makes complex programs manageable. OOP organizes code into objects that interact, promoting reusability, modularity, and maintainability. Unlike procedural programming, it focuses on "what" (objects) rather than "how" (steps).

The Four Pillars of OOP

  • Class: A blueprint for objects (e.g., a Car class defines properties like color and methods like drive).
  • Object: An instance of a class (e.g., myCar is an object of Car).
  • Encapsulation: Hiding data for security (using private fields and public methods).
  • Inheritance: Reusing code from parent classes (e.g., Dog inherits from Animal).
  • Polymorphism: One interface, many forms (e.g., method overriding for different behaviors).
  • Abstraction: Focusing on essentials (hiding complexity with abstract classes/interfaces).

Adventure Awaits: OOP lets you create game characters with unique abilities!

Benefits of OOP

Easier maintenance, code reuse, scalability, and real-world modeling. It reduces bugs by isolating changes.

When to Use OOP

For large projects, simulations, or anything with entities that interact (e.g., games, apps).

Examples and Code Snippets

Basic OOP structure.

// Class definition
class Animal {
  String name;
  void eat() {
    System.out.println(name + " is eating.");
  }
}

// Object creation
Animal dog = new Animal();
dog.name = "Buddy";
dog.eat();

Try It Yourself: OOP Exercises

Build and run these classes.

Exercise 1: Basic Class and Object

// Define class
class Player {
  String name;
  int health;

  void display() {
    System.out.println("Player: " + name + ", Health: " + health);
  }
}

// Main class
public class OOPDemo {
  public static void main(String[] args) {
    Player p1 = new Player();
    p1.name = "Hero";
    p1.health = 100;
    p1.display();

    Player p2 = new Player();
    p2.name = "Villain";
    p2.health = 80;
    p2.display();
  }
}
// Output:
// Player: Hero, Health: 100
// Player: Villain, Health: 80

Exercise 2: Encapsulation Example

// Encapsulated class
class BankAccount {
  private double balance;

  public void deposit(double amount) {
    if (amount > 0) balance += amount;
  }

  public double getBalance() {
    return balance;
  }
}

// Usage
public class EncapsulationDemo {
  public static void main(String[] args) {
    BankAccount acc = new BankAccount();
    acc.deposit(500);
    System.out.println("Balance: $" + acc.getBalance());
    // acc.balance = -100; // Error: private
  }
}
// Output:
// Balance: $500.0

Common Pitfalls and Tips

Avoid overusing inheritance; prefer composition. Use meaningful class names. In games, model entities like Player or Enemy as classes.
